### Obs Dimension Fix in Eval Function
- **Bug**: `_evaluate_head_to_head_full_obs()` silently dropped `partial_obs_los`, `sensing_radius`, `combined_masking`, `n_obstacles_min`, `n_obstacles_max`, `asymmetric_obs` via `**kwargs`
- **Symptom**: Model trained with 21-dim obs (partial obs enabled) but eval env created with 20-dim obs (partial obs disabled)
- **Fix**: Added all 6 missing parameters explicitly to the function signature and passed them to the `PursuitEvasionEnv` constructor

### SP4 Launch & Results
Launched SP4 on niro-2: combined partial obs (radius 3.0m + LOS), variable obstacles (0-3), cold start, 10M steps.

**SP4 Final Results**:
| Metric | Value |
|--------|-------|
| Total steps | 9,998,336 (10M) |
| Total time | 1.7h (99m) |
| Converged | No |
| Final SR_P | 0.39 |
| Final SR_E | 0.61 |
| Final NE gap | 0.22 |
| Min NE gap | **0.02** (M2750, ~5.6M steps) |
| Mean NE gap | 0.59 |
| NE gap trend | Decreasing |

**Best checkpoints**:
- Best pursuer: SR=0.53 at M2550 (5.2M steps)
- Best evader: SR=1.00 at M100 (204K steps)

**Phase breakdown**:
1. **0-2M**: Bootstrapping — pursuer stuck at 0-12%
2. **2-2.3M**: Breakout — pursuer jumps to 24% then 43%
3. **2.5-2.75M**: Near-equilibrium — SR_P hit 53% (gap=0.06), then 49% (gap=0.02)
4. **3-10M**: Oscillation — SR_P ranges 13-45%, typically 30-40%

**Key finding**: This is the **first successful cold-start self-play** in the project. All previous attempts (SP2/SP2b/SP2c/SP2d) failed with pursuer stuck at 0-7%. Combined partial obs + variable obstacles creates a bootstrappable game.
