### SP6b warm-seed crash analysis
- Error: `AttributeError: 'ActorCriticPolicy' object has no attribute 'actor'`
- Cause: Opponent pool loaded PPO checkpoint with `SAC.load()` — PPO's `ActorCriticPolicy` lacks SAC's `.actor` attribute
- Root: Mixed PPO/SAC warm-seeding is fundamentally incompatible at the policy architecture level
- Fix: Train SAC evader separately (RunC), then warm-seed with both SAC models

## Decisions Made
| Decision | Rationale |
|----------|-----------|
| Keep `ppo.zip` filename in checkpoints | Backward compatible with existing checkpoints; SB3 doesn't care about filename |
| `ent_coef="auto"` for SAC | SAC's automatic entropy tuning is superior to manual tuning |
| `learning_starts=1000` | Short warmup for self-play micro-phases (2048 steps each). Default SB3 is 100, but we want some warmup. |
| Mixed warm-seeding via `--init_pursuer_algo`/`--init_evader_algo` | Allows loading PPO evader (S1v6b) into SAC self-play session |
| `hasattr(opp, "policy")` instead of `isinstance(opp, PPO)` | Algorithm-agnostic: both PPO and SAC have `.policy` attribute |
| SAC net_arch [256,256] + Tanh | Match PPO architecture for fair comparison |
| Kill SP6a/SP6b cold starts | Both stalled at ~10% pursuer for 3M steps — cold-start SAC self-play can't bootstrap |
| Train SAC evader separately (RunC) before warm-seeded self-play | Need SAC-compatible evader model; can't load PPO S1v6b into SAC opponent pool |
| Kill SP7 (1:1 alternation), replace with SP7a+SP7b | Evader adapted too fast under equal alternation (20%→11%). Need asymmetric training. |
| SP7a: adaptive boost (loser gets 5 extra phases when gap>0.3) | Let the losing agent catch up dynamically |
| SP7b: freeze evader (pursuer-only training) | Give pursuer a stable learning target before enabling co-evolution. Most promising approach so far. |
| Reduce eval_interval_micro from 50 to 150 | Eval overhead (100 episodes) was slowing training. 3x less eval → ~175 steps/s (was 155). |
| Unfreeze at 42% pursuer SR | SP7b2 hit 42% — strong enough for co-evolution. Launched SP8 with full alternation. |
| RunD: 1.1x evader speed diagnostic | Test if smaller speed advantage forces obstacle exploitation. Result: still pure speed strategy (LOS 100% clear), but harder (2/6 escaped vs 5/6 at 1.2x). |
| Kill SP8 at 11% (1.3M steps) | Co-evolution with 1:1 alternation conclusively fails — evader adapts from 42% to 11% in one eval cycle. |
| Kill SP7a2 at 4.1M steps | Adaptive boost never sustained above 24%. Freed CPU for SP9. |
| 5:1 asymmetric train_ratio for SP9 | Give pursuer 5x more training than evader to slow evader's adaptation advantage. First results (30%→24%) much better than SP8 (11%). |
