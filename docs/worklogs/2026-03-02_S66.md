## Work Done

### SAC integration in `training/amsdrl.py`
- **Import**: Added `SAC` to `from stable_baselines3 import PPO, SAC`
- **Builder functions**: Added `_build_full_obs_sac()` and `_build_partial_obs_sac()` — mirror PPO builders but with SAC-specific params (buffer_size, learning_starts, tau, train_freq, gradient_steps)
- **`__init__` params**: Added `algorithm`, `buffer_size`, `learning_starts`, `sac_batch_size`, `tau`, `train_freq`, `gradient_steps`, `init_pursuer_algo`, `init_evader_algo`
- **Algorithm dispatch**: `self._model_class = SAC if algorithm == "sac" else PPO`, `_get_build_fn()` returns correct builder, `_get_build_kwargs()` returns PPO- or SAC-specific kwargs
- **`_warm_seed()`**: Rewritten to use `_get_build_fn()` and `_get_build_kwargs()`. Supports mixed algorithm loading via `init_pursuer_algo`/`init_evader_algo`. Hyperparam override is now algorithm-aware (PPO has `n_steps`, SAC does not).
- **`_cold_start()`**: Rewritten to use `_get_build_fn()` and `_get_build_kwargs()`. Full-obs and partial-obs both dispatched correctly.
- **PPO.load() → self._model_class.load()**: Replaced 5 call sites: `_warm_seed` (2), `_train_phase` (1), `_run_micro_phases` n_envs reload (1), collapse rollback (2)
- **isinstance(opp, PPO) → hasattr(opp, "policy")**: Updated `_sync_opponent_weights_vec()` to be algorithm-agnostic
- **pool.get_model()**: All 3 call sites now pass `model_class=self._model_class`
- **Verbose output**: Shows algorithm name in run() and _run_micro_phases() headers

## Decisions Made
| Decision | Rationale |
|----------|-----------|
| Keep `ppo.zip` filename in checkpoints | Backward compatible with existing checkpoints; SB3 doesn't care about filename |
| `ent_coef="auto"` for SAC | SAC's automatic entropy tuning is superior to manual tuning |
| `learning_starts=1000` | Short warmup for self-play micro-phases (2048 steps each). Default SB3 is 100, but we want some warmup. |
| Mixed warm-seeding via `--init_pursuer_algo`/`--init_evader_algo` | Allows loading PPO evader (S1v6b) into SAC self-play session |
| `hasattr(opp, "policy")` instead of `isinstance(opp, PPO)` | Algorithm-agnostic: both PPO and SAC have `.policy` attribute |
| SAC net_arch [256,256] + Tanh | Match PPO architecture for fair comparison |
