### `--freeze_switch_consecutive` Parameter
- Added to `scripts/train_amsdrl.py` and `training/amsdrl.py`
- Requires N consecutive evals above threshold before switching frozen role
- Prevents premature alternation from single lucky evals (SP11f's main failure mode)
- Streak resets to 0 if SR drops below threshold between evals

### Visibility-Gated Search Reward
- Modified `envs/pursuit_evasion_env.py`: search staleness reward (`w_search`) now only applies when evader is NOT visible to pursuer
- When evader is visible → zero search reward, capture/distance rewards drive pursuit
- When evader is hidden → search reward incentivizes systematic area coverage
- Uses same LOS + sensing radius check as `_get_obs()` (combined masking aware)
- SP11g's failure was that search reward was always-on, diluting the capture signal
