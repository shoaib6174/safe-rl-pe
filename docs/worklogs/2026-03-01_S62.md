## Work Done

### greedy_full_obs Implementation
- **Problem**: S1v5b (combined masking, 96% peak) escape rate is inflated because the greedy pursuer is also blinded by symmetric partial obs. When evader is out of sensing range (>3.0m), the greedy pursuer's obs has zeros for opponent position, causing it to steer toward center instead of chasing.
- **Fix**: Added `greedy_full_obs` param to `SingleAgentPEWrapper.__init__()`. When enabled and `role == "evader"`, the wrapper builds an unmasked observation for the opponent (pursuer) using `obs_builder.build(..., los_blocked=False)` before passing it to `opponent_policy.predict()`.

### CLI Integration
- Added `--greedy_full_obs` (store_true) arg to `train_evader_vs_greedy.py`
- Plumbed through `make_env()` and `evaluate()` functions
- Updated banner to show `Greedy obs: FULL (unmasked)` vs `Greedy obs: MASKED (same as evader)`
- Updated baseline loop to build unmasked obs for greedy pursuer when flag is set

### Tests
- `test_greedy_full_obs_sees_evader`: Verifies that with `greedy_full_obs=True`, the full obs built for the pursuer has non-zero opponent features even when evader is out of sensing range
- `test_greedy_default_masked_when_out_of_range`: Confirms default behavior — pursuer obs is masked when out of range without the flag

### S1v5 Series Final Results
All three runs completed on niro-2 (5M steps each, ~25-62 min):

| Run | Mode | Greedy Obs | Peak Escape | Final Escape | Time |
|-----|------|-----------|-------------|-------------|------|
| **S1v5** | Radius 3.0m (no LOS) | Masked | **91%** (3.6M) | 75% | 62m |
| **S1v5b** | Combined (3.0m + LOS) | Masked | **96%** (3.65M) | 84% | 62m |
| **S1v5c** | Combined (3.0m + LOS) | **Full (unmasked)** | **16%** (2.15M) | **0%** | 25m |

**Key finding**: S1v5b's 96% escape rate was ~80pp inflated by pursuer blindness. When the greedy pursuer always sees the evader (S1v5c), the evader peaks at only 16% and collapses to 0%. At equal speed with combined masking, the evader has no viable strategy against a full-obs pursuer.

**Baseline comparison**: S1v5c random evader baseline = 0.00 escape (vs likely higher in S1v5/S1v5b where pursuer was also blinded).

**S1v5c ran 2.5x faster** (25m vs 62m) because episodes terminated quickly (avg ~145 steps vs ~530 steps when evader escapes).
