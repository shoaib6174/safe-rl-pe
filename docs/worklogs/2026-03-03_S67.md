# Session S67 — Exploration Reward Attempted & Reverted

## Date: 2026-03-03
## Session Number: 67
## Previous Session: S66

## Context
SP11a/SP11b running with alternate_freeze on niro-2. Pursuer has 89% capture after detection but 43% miss rate. Considered adding exploration reward to incentivize search.

## Objectives
1. Add pursuer exploration reward
2. Evaluate whether it's the right approach

## Work Done
- [x] Implemented `--w_exploration` (grid-cell visitation bonus) across all 4 files
- [x] Tests passed (733, no regressions)
- [x] Pushed and launched SP12a/SP12b on niro-2
- [x] **Reverted** after analysis: agent lacks observational capacity to exploit exploration reward (no memory of visited cells in obs), and the reward doesn't account for evader movement (visited cells become stale)
- [x] Killed SP12a/SP12b, relaunched SP11a/SP11b to continue training longer

## Decisions Made
- Exploration reward is the wrong approach — the problem is observational, not motivational
- The existing distance-based reward already incentivizes finding the evader (capture = +5.0)
- Better to let SP11a/SP11b train longer than add reward complexity

## Key Insight
Hand-crafted exploration rewards add complexity without benefit when the agent can't observe the information needed to exploit them. The pursuer has no spatial memory in its observation — it can't know which areas it has or hasn't covered.

## Next Steps
- Monitor SP11a/SP11b for continued improvement with more training steps
